Create a palette with loon\textquotesingle{}s color mapping. 

\begin{DoxyParagraph}{Description}
Used to map nominal data to colors. By default these colors are chosen so that the categories can be well differentiated visually (e.\+g. to highlight the different groups) 
\end{DoxyParagraph}
\begin{DoxyParagraph}{Details}
This is the function that loon uses by default to map values to colors. Loon\textquotesingle{}s mapping algorithm is as follows\+:
\begin{DoxyItemize}
\item if all values already represent valid Tk colors (see tkcolors) then those colors are taken 
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyParagraph}{}

\begin{DoxyItemize}
\item if the number of distinct values is less than the number of values in loon\textquotesingle{}s color mapping list then they get mapped according to the color list, see l\+\_\+set\+Color\+List and l\+\_\+get\+Color\+List. 
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyParagraph}{}

\begin{DoxyItemize}
\item if there are more distinct values than there are colors in loon\textquotesingle{}s color mapping list then loon\textquotesingle{}s own color mapping algorithm is used. See loon\+\_\+palette and the details section in the documentation of l\+\_\+set\+Color\+List. 
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
For other mappings see the col\+\_\+numeric and col\+\_\+factor functions from the scales package.
\end{DoxyParagraph}
\begin{DoxyReturn}{Returns}
A function that takes a vector with values and maps them to a vector of 6 digit hexadecimal encoded color representation (strings). Note that loon uses internally 12 digit hexadecimal encoded color values. If all the values that get passed to the function are valid color names in Tcl then those colors get returned hexencoded. Otherwise, if there is one or more elements that is not a valid color name it uses the loons default color mapping algorithm. 
\end{DoxyReturn}

Convert 12 hexadecimal digit color representations to 6 hexidecimal digit color representations. 

\begin{DoxyParagraph}{Description}
Tk colors must be in 6 hexadecimal format with two hexadecimal digits for each of the red, green, and blue components. Twelve hexadecimal digit colors have 4 hexadecimal digits for each. This function converts the 12 digit format to the 6 provided the color is preserved.
\end{DoxyParagraph}

Use custom colors for mapping nominal values to distinct colors. 

\begin{DoxyParagraph}{Description}
Modify loon\textquotesingle{}s color mapping list to a set of custom colors.
\end{DoxyParagraph}

\begin{DoxyParams}{Parameters}
{\em colors} & list with valid color names or hex-\/encoded colors \\
\hline
\end{DoxyParams}
\begin{DoxyParagraph}{Details}
There are two commonly used mapping schemes of data values to colors one scheme maps numeric values to colors on a color gradient and the other maps nominal data to colors that can be well differentiated visually (e.\+g. to highlight the different groups). Presently, loon always uses the latter approach for its color mappings. You can use specialized color pallettes to map continuous values to color gradients as shown in the examples below. When assigning values to a display state of type color then loon maps those values using the following rules
\begin{DoxyItemize}
\item if all values already represent valid Tk colors (see tkcolors) then those colors are taken.
\item if the number of distinct values are less than number of values in loon\textquotesingle{}s color mapping list then they get mapped according to the color list, see l\+\_\+set\+Color\+List and l\+\_\+get\+Color\+List.
\item if there are more distinct values as there are colors in loon\textquotesingle{}s color mapping list then loon\textquotesingle{}s own color mapping algorithm is used. See loon\+\_\+palette and for more details about the algorithm below in this documentation. 
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
Loon\textquotesingle{}s default color list is composed of the first 11 colors from the hcl color wheel (displayed below in the html version of the documentation). The letters in hcl stand for hue, chroma and luminance, and the hcl wheel is useful for finding \char`\"{}balanced colors\char`\"{} with the same chroma (radius) and luminance but with different hues (angles), see Ross Ihaka (2003) \char`\"{}\+Colour for presentation graphics\char`\"{}, Proceedings of D\+SC, p. 2 (@url \href{https://www.stat.auckland.ac.nz/~ihaka/courses/787/color.pdf}{\texttt{ https\+://www.\+stat.\+auckland.\+ac.\+nz/$\sim$ihaka/courses/787/color.\+pdf}}). 
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
The colors in loon\textquotesingle{}s internal color list are also the default ones listed as the \char`\"{}modify color actions\char`\"{} in the analysis inspectors. To query and modify loon\textquotesingle{}s color list use l\+\_\+get\+Color\+List and l\+\_\+set\+Color\+List. 
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
In the case where there are more unique data values than colors in loon\textquotesingle{}s color list then the colors for the mapping are taken from different locations distributed on the hcl color wheel (see above). 
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
One of the advantages of using the hcl color wheel is that one can obtain any number of \char`\"{}balanced colors\char`\"{} with distinct hues. This is useful in encoding data with colors for a large number of groups; however, it should be noted that the more groups we have the closer the colors sampled from the wheel become and, therefore, the more similar in appearance. 
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
A common way to sample distinct \char`\"{}balanced colors\char`\"{} on the hcl wheel is to choose evenly spaced hues distributed on the wheel (i.\+e. angles on the wheel). However, this approach leads to color sets where most colors change when the sample size (i.\+e. the number of sampled colors from the wheel) increases by one. For loon, it is desirable to have the first m colors of a color sample of size m+1 to be the same as the colors in a color sample of size m, for all positive natural numbers m. Hence, we prefer to have a sequence of colors. This way, the colors on the inspectors stay relevant (i.\+e. they match with the colors of the data points) when creating plots that encode with color a data variable with different number of groups. 
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
We implemented such a color sampling scheme (or color sequence generator) that also makes sure that neighboring colors in the sequence have different hues. In you can access this color sequence generator with loon\+\_\+palette. The color wheels below show the color generating sequence twice, once for 16 colors and once for 32 colors. 
\end{DoxyParagraph}
\begin{DoxyParagraph}{}
Note, for the inspector\+: If there are more unique colors in the data points than there are on the inspectors then it is possible to add the next five colors in the sequence of the colors with the +5 button. Alternatively, the + button on the modify color part of the analysis inspectors allows the user to pick any additional color with a color menu. Also, if you change the color mapping list and close and re-\/open the loon inspector these new colors show up in the modify color list. 
\end{DoxyParagraph}
