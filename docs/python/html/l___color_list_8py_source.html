150 def l_setColorList(colors):
151  """
152  Use custom colors for mapping nominal values to distinct colors
153 
154  Description:
155  Modify loon's color mapping list to a set of custom colors.
156 
157  Args:
158  colors: list with valid color names or hex-encoded colors
159  Details:
160  There are two commonly used mapping schemes of data values to
161  colors: one scheme maps numeric values to colors on a color gradient and
162  the other maps nominal data to colors that can be well differentiated
163  visually (e.g. to highlight the different groups). Presently, loon always
164  uses the latter approach for its color mappings. You can use specialized
165  color pallettes to map continuous values to color gradients as shown in the
166  examples below.
167  When assigning values to a display state of type color then loon maps those
168  values using the following rules
169  - if all values already represent valid Tk colors (see
170  tkcolors) then those colors are taken.
171  - if the number of distinct values are less than number of values in
172  loon's color mapping list then they get mapped according to the color list,
173  see l_setColorList and l_getColorList.
174  - if there are more distinct values as there are colors in loon's color
175  mapping list then loon's own color mapping algorithm is used. See
176  loon_palette and for more details about the algorithm below
177  in this documentation.
178 
179  Loon's default color list is composed of the first 11 colors from the
180  hcl color wheel (displayed below in the html version of the
181  documentation). The letters in hcl stand for hue, chroma and luminance, and
182  the hcl wheel is useful for finding "balanced colors" with the same chroma
183  (radius) and luminance but with different hues (angles), see Ross Ihaka
184  (2003) "Colour for presentation graphics", Proceedings of DSC, p. 2
185  (https://www.stat.auckland.ac.nz/~ihaka/courses/787/color.pdf).
186 
187  The colors in loon's internal color list are also the default ones listed
188  as the "modify color actions" in the analysis inspectors. To query and
189  modify loon's color list use l_getColorList and l_setColorList.
190 
191  In the case where there are more unique data values than colors in loon's
192  color list then the colors for the mapping are taken from different
193  locations distributed on the hcl color wheel (see above).
194 
195  One of the advantages of using the hcl color wheel is that one can obtain
196  any number of "balanced colors" with distinct hues. This is useful in
197  encoding data with colors for a large number of groups; however, it should
198  be noted that the more groups we have the closer the colors sampled from
199  the wheel become and, therefore, the more similar in appearance.
200 
201  A common way to sample distinct "balanced colors" on the hcl wheel is to
202  choose evenly spaced hues distributed on the wheel (i.e. angles on the
203  wheel). However, this approach leads to color sets where most colors change
204  when the sample size (i.e. the number of sampled colors from the wheel)
205  increases by one. For loon, it is desirable to have the first m
206  colors of a color sample of size m+1 to be the same as the colors in
207  a color sample of size m, for all positive natural numbers m.
208  Hence, we prefer to have a sequence of colors. This way, the colors on the
209  inspectors stay relevant (i.e. they match with the colors of the data
210  points) when creating plots that encode with color a data variable with
211  different number of groups.
212 
213  We implemented such a color sampling scheme (or color sequence generator)
214  that also makes sure that neighboring colors in the sequence have different
215  hues. In you can access this color sequence generator with
216  loon_palette. The color wheels below show the color
217  generating sequence twice, once for 16 colors and once for 32 colors.
218 
219  Note, for the inspector: If there are more unique colors in the data points
220  than there are on the inspectors then it is possible to add the next five
221  colors in the sequence of the colors with the +5 button.
222  Alternatively, the + button on the modify color part of the analysis
223  inspectors allows the user to pick any additional color with a color menu.
224  Also, if you change the color mapping list and close and re-open the loon
225  inspector these new colors show up in the modify color list.
226 
227  When other color mappings of data values are required (e.g. numerical data
228  to a color gradient) then the functions in the scales
229 
230  Returns:
231  None
232  @see l_setColorList, l_getColorList, l_setColorList_ColorBrewer, l_setColorList_hcl
233  Examples:
234  @code
235  l_plot([1,2,3], color=[1,2,3]) # loon's default mapping
236  cols = l_getColorList()
237  l_setColorList(["red", "blue", "green", "orange"])
238  ## close and reopen inspector
239  l_plot([1,2,3], color=[1,2,3]) # use the new color mapping
240  l_plot(list(range(10)), color=list(range(10))) # use loons default color mapping as color list is too small
241  # reset to default
242  l_setColorList(cols)
243  @endcode
244  @namespace loon.l_setColorList
245  """
246  tk.tk.call('::loon::setColorList', 'custom', colors)
