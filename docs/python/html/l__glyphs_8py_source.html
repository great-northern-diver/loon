137 #' @title Add non-primitive glyphs to a scatterplot or graph display
138 #'
139 #' @description Generic method for adding user-defined glyphs. See details for
140 #' more information about non-primitive and primitive glyphs.
141 #' @family glyph functions
142 #'
143 #' @template param_widget
144 #' @param type object used for method dispatch
145 #' @param ... arguments passed on to method
146 #'
147 #' @details The scatterplot and graph displays both have the n-dimensional state
148 #' \code{'glyph'} that assigns each data point or graph node a glyph (i.e. a
149 #' visual representation).
150 #'
151 #' Loon distinguishes between primitive and non-primitive glyphs: the primitive
152 #' glyphs are always available for use whereas the non-primitive glyphs need to
153 #' be first specified and added to a plot before they can be used.
154 #'
155 #' The primitive glyphs are:
156 #'
157 #' \ifelse{html}{\figure{point_glyph_primitive_types.png}{options: alt="Primitive Glyphs"}}{
158 #' \tabular{l}{
159 #' \code{'circle'}, \code{'ocircle'}, \code{'ccircle'}\cr
160 #' \code{'square'}, \code{'osquare'}, \code{'csquare'}\cr
161 #' \code{'triangle'}, \code{'otriangle'}, \code{'ctriangle'}\cr
162 #' \code{'diamond'}, \code{'odiamond'}, \code{'cdiamond'}
163 #' }
164 #'
165 #' Note that the letter \code{'o'} stands for outline only, and the letter
166 #' \code{'c'} stands for contrast and adds an outline with the
167 #' \code{'foreground'} color (black by default).
168 #' }
169 #'
170 #' The non-primitive glyph types and their creator functions are:
171 #'
172 #' \if{html}{\figure{display_plot_glyphs_nonprimitive.png}{options: alt="Non-Primitive Glyphs"}}
173 #'
174 #' \tabular{ll}{
175 #' Type \tab R creator function\cr
176 #' Text \tab \code{\link{l_glyph_add_text}}\cr
177 #' Serialaxes \tab \code{\link{l_glyph_add_serialaxes}}\cr
178 #' Pointranges \tab \code{\link{l_glyph_add_pointrange}}\cr
179 #' Images \tab \code{\link{l_glyph_add_image}}\cr
180 #' Polygon \tab \code{\link{l_glyph_add_polygon}}
181 #' }
182 #'
183 #' When adding non-primitive glyphs to a display, the number of glyphs needs to
184 #' match the dimension \code{n} of the plot. In other words, a glyph needs to be
185 #' defined for each observations. See in the examples.
186 #'
187 #' Currently loon does not support compound glyphs. However, it is possible to
188 #' cunstruct an arbitrary glyph using any system and save it as a png and then
189 #' re-import them as as image glyphs using \code{\link{l_glyph_add_image}}.
190 #'
191 #' @templateVar page learn_R_display_plot
192 #' @templateVar section glyphs
193 #' @template see_l_help
194 #'
195 #'
196 #' @return String with glyph id. Every set of non-primitive glyphs has an id

199 #' @export

202 #'
203 #' @examples
204 #' # Simple Example with Text Glyphs
205 #' p <- with(olive, l_plot(stearic, eicosenoic, color=Region))
